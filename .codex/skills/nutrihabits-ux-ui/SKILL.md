# NutriHabits UX/UI

Read `docs/PRODUCT_SCOPE.md`, `docs/ROADMAP.md`, and `docs/subagents/ux-auditor.md`.

## Focus

- nutritionist backoffice clarity
- patient mobile-first usability
- truthful UI over demo placeholders
- explicit empty, pending, and error states

## Workflow

1. Read the current controller and view together.
2. Replace fake metrics or dead links before polishing visuals.
3. Preserve the user flow intent even if markup changes.
4. Note UX-impacting decisions in the worklog.
